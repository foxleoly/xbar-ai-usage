# TokenDock Handoff

Date: 2026-05-23

## Goal

Build TokenDock: an iPhone and Apple Watch app that shows token usage for coding agents. The intended data path is:

Mac daemon -> Bluetooth LE -> iPhone app -> WatchConnectivity -> Apple Watch app

The user explicitly prefers Bluetooth over WebSocket for Mac-to-iPhone transport.

The chosen app name is TokenDock.

The iPhone app should eventually include a charging display mode: when the phone is charging, it can work like a small desk display / screen-saver ornament that shows token usage at a glance.

The current iOS app icon direction is "H / Token Rings": a dark Apple-style icon with teal token rings. The icon has been generated and wired into the Xcode asset catalog.

## Current Status

### Mac Daemon

Path: `mac-daemon/`

Implemented:

- Reads Codex token usage from `~/.codex/state_5.sqlite` and rollout JSONL files.
- Builds an `agent_usage_snapshot` JSON payload.
- Supports usage windows:
  - Rolling 5 hours
  - Today
  - 7 days
  - 30 days
- Advertises usage snapshots over BLE as a peripheral.
- BLE service UUID: `8B7D2F4E-9678-4A13-A890-A5E89D7D6C01`
- BLE snapshot characteristic UUID: `0A5B95C6-20B7-4149-8E26-AB0BD034A07C`
- BLE wire chunks use a 9-byte header:
  - Magic: `0x41 0x55`
  - Version: `0x01`
  - Message ID: UInt16 big-endian
  - Chunk index: UInt16 big-endian
  - Chunk count: UInt16 big-endian
